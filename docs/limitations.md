# 限制说明

## 数据边界

- 只读取当前扫码账号在抖音创作者中心页面真实可见的数据。
- 页面没有展示的字段保存为 `null`，不会使用 0、历史值或模型推测填充。
- 不调用、逆向或绕过未公开私有 API。
- 列表指标与详情指标保留独立来源；同名字段不静默覆盖。
- 详情同步写入前必须确认目标作品。没有稳定详情链接、平台 ID 且标题也无法匹配时，返回 `video_identity_unresolved`。

## 平台变化

抖音创作者中心是持续变化的网页应用。DOM、文案、指标可见性和风控策略变化时，可能出现：

- `partial`：只采集到部分作品或部分字段；
- `parser_degraded`：页面存在，但当前解析器没有识别到可信指标；
- `user_action_required`：登录过期或出现安全验证，需要用户在可见 Chrome 中处理；
- `profile_in_use`：另一个进程正在使用专用 profile。

这些状态不会覆盖已有可信快照。

## 详情指标真实验证

详情页解析、身份校验、分批游标、缓存、快照和派生公式已有自动化测试。由于不同账号的创作者中心功能与字段可能不同，发布前仍需在目标账号运行真实详情 smoke，逐项核对页面显示值与 SQLite 快照。

历史上作品列表已用真实账号验证页面声明总数、加载数和解析数一致。真实作品数量与指标不保存在仓库；该结果也不代表所有账号或未来页面版本都天然可用。

## 分析限制

- 周期参数按本地快照采集时间筛选，不等同于平台自然日归因窗口。
- 潜力排序是本地透明启发式，不是抖音官方评分；样本少于 10 条时只供参考。
- 没有足够详情覆盖率时，Agent 必须降低结论置信度并明确缺失字段。
- V1 不建设多账号、云端调度、复杂任务队列或团队权限系统。
- CSV/Markdown 公式注入转义已按产品决策延后，不应将导出内容交给不可信来源自动执行。

## 安全边界

- 不保存账号密码，不向 Agent 返回 Cookie、localStorage、sessionStorage 或验证码。
- profile 和 SQLite 只保存在本地专用目录，不应提交 Git 或暴露到公网。
- 首次成功同步会建立不可逆的轻量账号指纹；后续列表同步无法匹配时，在写入前返回 `account_mismatch`。
- 账号指纹只保存加盐哈希和安全状态，不向 Agent 返回作品标题、作品 ID、随机盐、本机路径或进程信息。
- HTTP 模式必须配置 `MCP_HTTP_API_KEY`；默认推荐 stdio。
- 不要把 `DOUYIN_BROWSER_PROFILE_DIR` 指向日常 Chrome 的默认用户目录。
