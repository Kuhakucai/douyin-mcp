# 单账号轻量 V1 实施状态

## 已完成

- 默认 browser-only 容器，无 OpenAPI 密钥可启动；历史 OpenAPI 容器显式保留。
- MCP 工具固定内部账号 `browser-default`，不暴露账号选择参数。
- 统一 `douyin-mcp` CLI：init、doctor、login、status、sync、details、videos、performance、export、purge。
- 跨进程 profile 文件锁、死亡 PID/损坏 stale lock 自动恢复和确认式本地数据清除。
- 旧 SQLite 自动备份、幂等迁移、busy timeout、同步任务进度和游标。
- 虚拟滚动列表增量收集、来源指纹、parser version、完整性和字段覆盖率。
- 详情页指标保守提取、作品身份门禁、5～10 条分批、24 小时缓存和断点游标。
- append-only 原始快照、独立派生指标、公式版本、缺失原因和质量状态。
- 单作品查询、批量对比、覆盖率、轻量潜力排序、复盘上下文和 JSON/CSV 导出。
- 自动化测试覆盖迁移、解析、服务、工具、CLI、会话、锁定流程与公式。
- 首次列表同步建立不可逆轻量账号指纹，误切账号时在快照和业务数据写入前拒绝同步。

## 真实账号验收结论（2026-07-13）

- 作品列表同步的声明数量、加载数量和解析数量一致；真实数量不写入仓库。
- 无详情 URL 的列表卡片可用完整标题+北京时间发布时间唯一定位；打开后绑定 `/work-detail/{id}`，后续直接访问。
- 页面实际展示的详情指标已逐字段核对，原始显示值与标准化入库值一致；真实指标不写入仓库。
- 一批可查看详情的作品已完成采集与身份绑定，播放、点赞、收藏、评论、分享均按页面实际展示记录。
- 小批量与扩大批量的缓存复跑均无失败，游标和 24 小时缓存生效，没有重复采集。
- 页面未展示的字段保持 `null/not_displayed`，不再因为缺少某个固定字段（例如曝光量）把已展示字段全部解析成功的快照降级为 partial。
- 当前作品状态不支持详情时，系统返回 `data_not_available`，不写入伪造或空详情快照。
- 自动化回归、compileall、diff check、doctor 均通过；具体测试数量以当前测试输出为准。

真实账号逐值验收证据只保存在本地忽略文件 `docs/real-account-acceptance-report.md`，不得提交 Git。

## 后续观察项

1. 页面 DOM 或指标文案变化时需要升级 parser version；详情缓存只复用同版本快照。
2. `background_first` 的长期成功率和连续 50 条访问的风控表现仍需在日常使用中观察。
3. 作品首次绑定前若标题或发布时间发生变化，系统会安全失败；重新同步列表后再试即可。
