# 需求背景：浏览器真实联调入口（第三阶段）

## 用户目标

继续浏览器登录态主线，使开发者和本地 Agent 能以可重复方式验证以下链路：打开抖音创作者中心、复用本地 profile、判断登录状态、同步页面快照、生成报告、查看最近一次同步摘要。

## 当前基础

- 第一阶段已实现 Playwright 持久化 profile 会话层。
- 第二阶段已实现页面抽取器、`BrowserService`、四个 MCP browser tools、SQLite 快照和基础报告。
- 自动化基线为 `python -m pytest -q`：31 passed。
- 尚未对真实 Chrome 和真实抖音创作者中心页面完成可重复的手动联调。

## 本阶段约束

- 官方 OpenAPI、小程序和手动导入均不在当前开发主线。
- 不读取、输出或保存 Cookie、localStorage、sessionStorage、验证码和密码。
- 浏览器默认可见；登录或风控验证必须由用户在浏览器中完成。
- CLI 每次调用都应能独立启动并复用同一 profile，满足第二天本地 Agent 独立同步的场景。
- 不在缺少真实页面样本时臆造稳定 DOM 选择器或精准视频指标映射。
- 保留当前未提交改动，不回退用户或前序阶段文件。

## 成功标准

1. 提供一个安装后可直接运行的浏览器联调命令。
2. 支持登录、状态检查、同步、生成报告、查看安全快照摘要。
3. 命令输出稳定 JSON，适合人工检查和 Agent 调用。
4. 登录流程能等待用户处理扫码/验证，并在成功或超时后干净关闭浏览器。
5. 单元测试不启动真实浏览器，完整覆盖命令分发和敏感数据边界。
6. README、浏览器通道文档和 Agent 交接文档同步更新。
