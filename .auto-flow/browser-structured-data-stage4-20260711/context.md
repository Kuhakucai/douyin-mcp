# 需求背景：浏览器结构化视频数据（第四阶段）

## 用户目标

继续浏览器登录态主线，使本地 Agent 不仅能触发页面快照，还能获取用户自己的结构化作品数据，包括标题、发布时间、时长、播放、点赞、评论、分享等，并持久化到 `videos`、`video_metrics`。

## 当前基础

- 第一阶段已实现 Playwright 持久化 profile。
- 第二阶段已实现 MCP 浏览器工具、页面快照和基础报告。
- 第三阶段已实现真实联调 CLI，完成真实 Chrome 扫码登录、profile 复用、同步和报告验证。
- 自动化基线为 44 项测试通过。
- 真实页面结构已完成只读探测：
  - 页面声明的作品总数属于真实账号本地验收数据，不写入仓库。
  - 首屏加载后继续滚动，实测可加载到页面声明的全部作品。
  - 每个作品卡片有独立标题、发布时间、状态、时长及指标标签/值节点。
  - DOM 未暴露稳定的官方 item ID 或作品链接。

## 约束

- 只使用用户可见的创作者中心 DOM，不调用或逆向私有内部 API。
- 不读取、输出或保存 Cookie、Storage、验证码和密码。
- 不提交真实页面全文、真实作品标题、profile、数据库或报告到 Git。
- 页面类名带构建哈希，解析必须优先依赖结构和稳定类名前缀，并提供保守失败结果。
- 无官方作品 ID 时使用确定性本地 ID；必须保证重复同步不会重复插入视频或同日指标。
- 所有数据工具默认只读，并限制分页大小。
- 保留当前未提交改动，不回退前序阶段内容。

## 成功标准

1. 同步时自动滚动加载当前作品列表中的全部卡片。
2. 将每个卡片解析为标准字段并写入 `videos`、`video_metrics`。
3. 重复同步保持视频和同日指标幂等。
4. 新增 MCP 工具和 CLI 子命令，Agent 可分页获取结构化视频及最新指标。
5. 快照和报告反映结构化视频数量与指标汇总。
6. 自动化测试覆盖数值、日期、时长解析、滚动收敛、落库幂等、分页和隐私边界。
7. 真实 Chrome 验证能够同步并读取当前账号结构化视频。
